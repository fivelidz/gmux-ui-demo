Key Design Decisions

Extended Canvas Zone

  • Problem: MediaPipe gives [0,1] normalized coords. Hands near the frame edge disappear abruptly when they cross the boundary.
  • Solution: Remap landmark coordinates from [0,1] to [-0.12, 1.12] before passing to the renderer. This expands the drawn area 12% on all sides — hands tracked at the edge are drawn slightly outside the visible frame, so their disappearance is gradual, not a cliff.
  • Visual cue: A dashed border ring around the camera PiP activates when a hand wrist is within 10% of the edge — indicates "extended zone active".
  • Used in: Demo 01 (full screen) and Demo 03 (PiP panel).

Gesture → Voice Target Flow

  • Selecting an agent (swipe / click) immediately sets that agent as the voice target
  • Left POINT toggles mic — voice goes to whoever is selected
  • Left THREE jumps to the next waiting or permission agent and targets that one
  • Visual: sidebar row gets a red tint + mic icon, card gets a red top bar sweep animation

State Colour System

  • State is communicated via colour only, never via text alone — the coloured dot is readable from 3m on a projector
  • Cards use box-shadow: 0 0 0 1px rgba(color, 0.x) glow — strong enough to read across the room, subtle enough not to scream
  • Permission state pulses (CSS animation) — it's the only state that demands immediate attention
  • Working state has an animated dot ring (ripple expand) — distinct from the static dots of other states
